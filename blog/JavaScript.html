Two Libraries, One Language

Let's start with discussing exactly what libraries are in regards to programming. Libraries are what we call lists, in JavaScript and Ruby, that can be called upon to return corresponding values. Let's begin with Ruby Hashes as you may be a bit more familiar with those. As you can see at the top of your map we have a list of hopeful vacation destinations for your average 20 something independent music fan. Our hash is created when we name it vacation followed by the equal sign and curly brackets that hold the code. Inside of the curly brackets are the keys and their corresponding values. Now it's good to note that a key can have many values (i.e. each of these values being a small list), but a value can only belong to one key. In this example the keys and values are both strings, and thus we use the string "fall" to return the values ["Afropunk", "Brooklyn"] from the vacation hash.

On that same tangent, lets move on to our next Ruby Hash example. Here we have your standard Chicagoan's list of favorite sports teams. These keys all have one corresponding value. But there's something else, different here, can you tell what it is? With our favorite_teams hash we see that we can name as well as call your Ruby Hash Keys with symbols. For example if you'd like to know this example's favorite :football team you'd use the symbol :football with your favorite_teams hash. If we were to print favorite_teams[:football] it would return the string "Bears". So just a recap, Ruby Hashes have a key/value relationship. Where keys can be named using strings and symbols, and there can be multiple values for each key.

Next Let's discuss what Ruby Hashes have in common with JavaScript Objects. JavaScript Objects and Ruby Hashes: are both libraries, they store information based on a keyword, and they're constructed and called pretty similarly. Both the anatomy of the Hash and JavaScript Object both are named, followed by and equal sign and curly brackets that encapsulate their code. Both of these libraries also have the ability to run loops that allow you to do things with the lists. For example, both the Ruby (.each) and JavaScript (for) allow you to loop through all of the values in the respective libraries and do something with them under certain parameters. And although they have that, and many other things in common, you'll see, there's quite a few differences.

Although, JavaScript Objects may look similar to Ruby Hashes even those similarities have different names. While Hashes have keys and value pairs, JavaScript Objects have property and value pair. Now both of them use the word "value", which makes perfect sense you want to have a value returned to you. Also, although you can only use strings for properties you can call those properties of the in two different ways. These two different ways of summoning information from our are called dot notation and bracket notation (for semi-obvious reasons, just look long enough). When using the bracket notation your property can be anything as long as you call it using quotation marks to signify that it is a string. Also, just for reference, bracket notation works solely because no matter what you name your property it's turned into a string using the toString method. Now dot notation must have a property that is a valid JavaScript identifier i.e. a sequence of alphanumerical characters, also including the underscore ("_") and dollar sign ("$"), that cannot start with a number. When taking these two options into consideration, understand that this isn't like Hashes where you can use symbols and strings interchangeably. The dot notation is preferred because it works in compliance with JavaScripts identifiers, and the bracket notation should only be used if you absolutely must use a property that is not a JavaScript Identifier. Other differences you might notice, when you create a JavaScript Object it is prefaced with a "var" which is essentially a JavaScript identifier letting you know that hash is a variable and that variable's name is also the hash name.

Those are the basics. Let's have a quick walk recap. JavaScript Objects and Ruby Hashes are libraries constructed similarly with a lot of the same functions, and a few things named differently. Where Ruby Hashes have Keys, have properties. The keys in Ruby Hashes can be named using either strings or symbols as you please. can be called using either dot or bracket notation; however, dot notation is preferred, and bracket should only be used in situations where it is needed. In the end though, they're both libraries, and extremely useful.
