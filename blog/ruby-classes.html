The Road To Ruby

Today we'll be discussing best practices to writing Instance Methods, Local Variables and Instance Variables Instead of having this page be extremely long, I'd like to you click here: Ruby Die Class to view the example we'll be using today.

So what we're trying to do today, is find the most effective ways to set up variables and methods throughout within our class. We're going to use a die (singular of dice) in order to illustrate this example.

Let's start with Instance Methods. Our Instance Methods in this example(because they don't have to be but in this case are) held within our class, Die class. Everything we'll be using today will be within our instance methods. Our instance methods are the two white bolded words within the die class. You can usually tell an instance method as it is named right after the word "def" in Ruby. There's also another bolded "roll" outside of the Die class. That is not an Instance Method. Instead, after we've called a new Die class, that's what we use in order to call the Instance Method. If you don't call the Instance Method on the new occurance, it will never happen.

Within those Instance Methods are Local Varibles and Instance Variables. As you can guess they're both variables, but where they differ is their range in terms of where they are able to be called within the class and how they're coded.

Local Variables are variables that can be called anywhere within their Instance Method. For example the mint green "sides" inside of the initialize Instance Method is a Local Variable. It's actually the only one that we have in this example, and there's reason for that. Local Variables have names that start with a lower case letter or the underscore character (_) and can only be used within that class/scope. Another interesting thing about Local Variables is that they do not have the value "nil" before initialization. If you ask for their value before they're given a value at initialization it will return an error. Now I'm sure you're wondering why the "sides" inside of the parenthesis next to the initialize Instance Method isn't a Local Variable. That sides is actually used as the argument for the initialize Instance Method. It lets us know the number of, and what the arguments are for the Instance Method. For example if the parenthesis read (sides, color) it would let us know that the instance method is taking two different arguments, one for sides, and one for color (which I haven't accounted for).

So I know you're like, what about all the other "sides" in the Die class that start with @? The pink "@sides" are all, you guessed it Instance Variables. Instance Variables are named the same as Local Variables except before the the lower case letter or underscore there's an @ symbol. The @ symbol lets us know that it's an Instance Variable. Besides having the cool @ symbol there's more benefits to Instance Variables. Instance Variables have the ability to be called anywhere within the scope of the class. So you can call an Instance Variable in any Instance Method within a class. So I'm sure you're wondering, why not just make everything an Instance Variable? Well with the initialize Instance Method you actually need that Local Variable "string" in order to get the information from the argument, to place into the Instance Variable "@string". So, although all variables are not created equal, they should all be appreicated just the same.

So I know we've gone over a lot today, but I'm sure if you don't know now, when you understand how vital Instance Methods, Local Variables and Instance Variables will to your coding, you'll appreciate it.
